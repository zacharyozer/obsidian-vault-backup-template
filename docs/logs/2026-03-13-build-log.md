## Phase 3: Test headless sync

We had four questions to answer before building the workflow.

**Q3: Does sync-setup + sync work?**
```bash
ob sync-setup --vault "<test-vault>" --path /tmp/test-vault \
    --password "$E2E_PASSWORD"
ob sync --path /tmp/test-vault
ls /tmp/test-vault/  # plaintext files appeared
```
Result: **Yes.** Files synced and decrypted to plaintext.

**Q4: Is sync-setup idempotent?**
```bash
# Ran the exact same setup a second time
ob sync-setup --vault "<test-vault>" --path /tmp/test-vault \
    --password "$E2E_PASSWORD"
ob sync --path /tmp/test-vault
```
Result: **Yes.** Showed a merge warning but succeeded. Safe to run
every invocation -- no need to cache state between CI runs.

**Q5: What config files does sync-setup create?**
```bash
find /tmp/test-vault/.obsidian -type f 2>/dev/null
# Nothing meaningful in the vault dir
ls ~/.obsidian-headless/sync/
# Config stored here: config.json (with derived encryption key),
# state.db, sync.log
```
Result: Config lives in `~/.obsidian-headless/sync/<vault-id>/`, NOT
in the vault directory. The vault's `.obsidian/` stays empty.

**Q6: Does E2E password need to be passed every sync?**
```bash
# Tried syncing without --password after initial setup
ob sync --path /tmp/test-vault
```
Result: **No.** Password is only needed at `ob sync-setup`. It derives
an encryption key and stores it in
`~/.obsidian-headless/sync/<vault-id>/config.json`. Subsequent syncs
decrypt transparently using the stored key.

### Verification steps

After run 5 succeeded, the agent verified the pipeline end-to-end:

```bash
# Are files in the repo?
gh api repos/<user>/obsidian-vault-backup/contents/vault \
    --jq '.[].name'

# Are they encrypted on GitHub? (should be binary/gibberish)
gh api 'repos/<user>/obsidian-vault-backup/git/trees/main?recursive=1' \
    --jq '.tree[] | select(.path | startswith("vault/")) | .path' \
    | head -5

# Can we decrypt locally?
cd /tmp && gh repo clone <user>/obsidian-vault-backup test-clone
cd test-clone
git-crypt unlock /tmp/git-crypt-key.bin
cat vault/*.md  # plaintext confirmed
```

Files on GitHub showed `GITCRYPT` binary header. After clone + unlock,
plaintext notes were readable.

### Key findings

* **Concurrent sync:** Obsidian Sync allows one active connection per vault.
  If the desktop or mobile app is connected, the headless client gets
  "Another sync instance already running." Workflow handles gracefully
  with retry + skip. In practice, most hourly runs succeed because the
  desktop app only holds the connection briefly during active syncs.
* **git-crypt ordering:** `git-crypt unlock` must happen before `ob sync`.
  Unlock rewrites tracked files in place and requires a clean tree.
  Sync writes new files into `./vault/`, dirtying the tree.
* **sync-setup is idempotent:** Safe to run every invocation. No caching
  of `~/.obsidian-headless/` needed between runs.
* **Auth token as sole auth:** `OBSIDIAN_AUTH_TOKEN` env var sufficient --
  no `ob login` needed when the token is valid.
* **TOTP fallback works:** Deliberately invalidated token in CI,
  password+TOTP login succeeded. Both auth paths are now tested.
* **npm global vs local:** `npm install -g` on GH Actions puts modules
  in a global prefix that `node -e` can't resolve from the repo working
  directory. Local install (`npm install` without `-g`) puts modules in
  `./node_modules/` which Node finds automatically.
* **git-crypt verified:** Files on GitHub show `GITCRYPT` header. After
  clone + unlock locally, files are plaintext.
* **Commit step is gated:** `if: steps.sync.outputs.synced == 'true'`
  prevents committing when sync was skipped due to concurrent connection.
